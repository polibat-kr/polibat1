AS-IS vs TO-BE 비교

구분 AS-IS (현재) TO-BE (목표) 개선 효과
Backend API 없음 (Frontend Only) Node.js + Express 실제 데이터 처리, RESTful API 제공
Database localStorage PostgreSQL 16 + Redis 7 데이터 영속성, 동시성 제어, 캐싱
개발 환경 N/A 하이브리드 환경 PostgreSQL 16: 원격 서버, Redis 7: Docker
인증/인가 localStorage 기반 JWT + Refresh Token 보안 강화, 세션 관리, 토큰 갱신
상태 관리 Local useState Zustand + TanStack Query 전역 상태 관리, API 캐싱, 성능 향상
타입 시스템 부분 TypeScript Full TypeScript 타입 안정성, AI 개발 효율 극대화
프로젝트 구조 분리된 2개 앱 모노레포 (Turborepo) 코드 공유, 통합 관리, 일관성
AI 기능 불가능 MVP + 확장 3줄 요약, 감정 온도계, 정치 브리핑

다음 단계 & 타임라인

Week 1: AWS 인프라 기본 설정

AWS 계정 설정, IAM 사용자 생성, 도메인 등록 (polibat.com), SSL 인증서 발급 (ACM)

Week 2: 개발 환경 구축

PostgreSQL 16 원격 서버 구축 (43.201.115.132), Docker Desktop 설정, Redis 컨테이너 구성, 개발자 PC 환경 설정

Week 3: CI/CD 파이프라인

GitHub Actions 워크플로우 설정, ECR 리포지토리 생성, ECS 클러스터 및 서비스 생성

Week 4: 모니터링 설정

CloudWatch 알람, 팀즈 웹훅 통합, 로그 중앙 집중, 메트릭 대시보드 구축
